üöÄ SETUP INSTRUCTIONS
====================

OPTION 1: LOCAL MONGODB (Windows/Mac/Linux)
-------------------------------------------

1. Download and Install MongoDB Community Edition
   üì• Visit: https://www.mongodb.com/try/download/community
   - Choose your operating system
   - Follow installation wizard
   - Start MongoDB service

2. Open Command Prompt/Terminal and verify installation:
   mongod --version

3. Keep MongoDB running in background

OPTION 2: MONGODB ATLAS (Cloud - Recommended for beginners)
-----------------------------------------------------------

1. Create Free Account:
   üëâ Visit: https://www.mongodb.com/cloud/atlas
   - Sign up with Google, GitHub, or email
   - Create a free tier cluster (M0 - always free)

2. Create Connection String:
   - Click "Connect" on your cluster
   - Choose "Connect your application"
   - Copy the connection string
   - Replace <username> and <password> with your credentials
   - Example: mongodb+srv://user:password@cluster.mongodb.net/calendar_db

3. Update .env file:
   - Open: .env in your calendar folder
   - Replace MONGODB_URI value with your connection string

üìù CONFIGURE .ENV FILE
======================

The .env file in your calendar folder has these settings:

# For LOCAL MongoDB:
MONGODB_URI=mongodb://localhost:27017/calendar_db

# For MONGODB ATLAS (Cloud):
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/calendar_db

# Session secret (change this to random string in production)
SESSION_SECRET=your-super-secret-session-key-change-in-production

# Server port
PORT=3000

# Node environment
NODE_ENV=development

üîÑ KEY DIFFERENCES FROM JSON DATABASE
=====================================

OLD (JSON):                          NEW (MongoDB):
- Synchronous operations            - Async/await operations
- Files stored on disk (.json)       - Data in MongoDB cloud/local
- Limited scaling                    - Scales for thousands of users
- Manual session management          - Built-in session store
- File-based reporting               - Query-based reporting

üóÑÔ∏è DATA COLLECTIONS IN MONGODB
==============================

MongoDB automatically creates these collections:

1. users - Stores user accounts
   - username, password, role, email, name, createdAt

2. reports - Stores event reports
   - eventDate, eventName, reportTitle, reportContent, submittedBy

3. events - Stores calendar events
   - eventDate, title, description, type, createdBy

4. sessions - Stores user session data
   - Managed automatically by connect-mongo

üõ†Ô∏è TROUBLESHOOTING
==================

‚ùå "Cannot connect to MongoDB":
   ‚úì Make sure MongoDB service is running
   ‚úì Check MONGODB_URI in .env file
   ‚úì Verify MongoDB credentials if using Atlas

‚ùå "Error: listen EADDRINUSE :::3000":
   ‚úì Another app is using port 3000
   ‚úì Change PORT in .env file to 3001, 3002, etc.
   ‚úì Or stop the other application

‚ùå "Unique index violation on username":
   ‚úì User already exists in database
   ‚úì Try a different username

‚ùå "Connection timeout":
   ‚úì MongoDB Atlas: Check IP whitelist (allow all IPs: 0.0.0.0/0)
   ‚úì Check internet connection
   ‚úì Verify MONGODB_URI is correct
